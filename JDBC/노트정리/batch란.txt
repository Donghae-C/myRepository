배치(Batch)란?
 여러 개의 SQL 문을 한 번에 모아서(DBMS에게) 실행하는 방식
- 일반적인 JDBC 실행 방식은 SQL 한 개당 DB와 통신 → 네트워크 비용이 크고 성능 저하 발생
- Batch를 사용하면 다수의 SQL을 모아 한 번에 전송 → 성능 향상

JDBC에서는 addBatch()를 통해 여러 개의 SQL 문장을 하나의 묶음(batch) 으로 모아두고,
 executeBatch()를 호출하면 그 묶음 전체를 한꺼번에 실행한다.

왜 배치를 쓰는가?
1)성능 향상
매번 DB에 연결해서 처리하는 것보다, 한 번에 여러 건을 처리하면 훨씬 빠르고 효율적이다.

2)트랜잭션 관리 용이
전체 배치를 하나의 트랜잭션으로 묶어서 성공/실패를 함께 관리할 수 있다.

3)코드 간결화
반복문 안에서 매번 executeUpdate() 대신 addBatch()로 모아두고 한 번만 실행한다.
